java语言特点

  • 简单性：语法基于C/C++，去除了指针、多重继承等复杂特性，自动垃圾回收。
  • 面向对象：纯面向对象语言，一切皆对象（除基本数据类型）。
  • 平台无关性：编译成字节码(.class)，通过JVM实现“一次编写，到处运行”。
  • 安全性：无指针，有字节码验证机制。
  • 健壮性：强类型机制、异常处理、自动垃圾回收。
  • 多线程：内置多线程支持，可开发高并发程序。
  • 分布式：支持网络编程，可通过URL访问网络资源。

标识符规则

  • 由字母、数字、**下划线(_)、美元符号($)**组成。
  • 不能以数字开头。
  • 区分大小写。
  • 不能是Java保留关键字。

类的三大特性

封装性

将类的状态（成员变量）隐藏起来，仅通过公共方法（get/set）对外提供访问和修改，实现 “高内聚、低耦合”。

代码示例

class Person {


    // 私有成员变量（封装核心：禁止直接访问）


    private String name;


    private int age;




    // 公共构造方法


    public Person(String name, int age) {


        this.name = name;


        this.setAge(age); // 通过setter校验赋值


    }




    // get方法：对外读取数据


    public String getName() {


        return name;


    }




    // set方法：对外修改数据，可加入校验逻辑


    public void setAge(int age) {


        if (age < 0 || age > 120) {


            throw new IllegalArgumentException("年龄不合法");


        }


        this.age = age;


    }




    public int getAge() {


        return age;


    }


}




// 测试类


public class EncapsulationDemo {


    public static void main(String[] args) {


        Person p = new Person("张三", 25);


        System.out.println(p.getName() + "，年龄：" + p.getAge());


        p.setAge(30); // 合法修改


        // p.age = -5; // 编译报错，无法直接访问私有变量


    }


}

继承性

子类（派生类）继承父类（基类）的属性和方法，实现代码复用，同时可扩展新功能。Java 中使用 extends 关键字，且仅支持单继承。

代码示例

// 父类：动物类


class Animal {


    protected String name;




    public Animal(String name) {


        this.name = name;


    }




    public void eat() {


        System.out.println(name + " 正在吃饭");


    }


}




// 子类：狗类，继承Animal


class Dog extends Animal {


    public Dog(String name) {


        super(name); // 调用父类构造方法


    }




    // 子类特有方法


    public void bark() {


        System.out.println(name + " 汪汪叫");


    }


}




// 测试类


public class InheritanceDemo {


    public static void main(String[] args) {


        Dog dog = new Dog("旺财");


        dog.eat();  // 继承自父类的方法


        dog.bark(); // 子类特有方法


    }


}

多态性

同一行为，在不同子类中表现出不同的实现形式。核心依赖方法重写（Override）和父类引用指向子类对象。

代码示例

// 父类：形状类


abstract class Shape {


    // 抽象方法：无具体实现，由子类重写


    public abstract void draw();


}




// 子类：圆形


class Circle extends Shape {


    public void draw() {


        System.out.println("绘制圆形");


    }


}




// 子类：矩形


class Rectangle extends Shape {


    public void draw() {


        System.out.println("绘制矩形");


    }


}




// 测试类


public class PolymorphismDemo {


    public static void main(String[] args) {


        // 父类引用指向子类对象（多态核心）


        Shape s1 = new Circle();


        Shape s2 = new Rectangle();




        s1.draw(); // 输出：绘制圆形（调用Circle的实现）


        s2.draw(); // 输出：绘制矩形（调用Rectangle的实现）


    }


}

接口（interface）

  • 接口是特殊抽象类型，定义一组行为规范。
  • 定义：interface 接口名 { 抽象方法声明; }
  • 实现：class 类名 implements 接口名 { 实现所有抽象方法; }

接口与类共同点 - 都可编译成 .class 字节码文件 - 接口中的成员方法也是抽象方法

接口与类不同点 - 接口不能实例化 - 接口方法默认 public abstract - 接口变量默认 public static final 常量 - 接口支持多实现，弥补单继承限制 - 接口之间可以多继承

接口成员特点 - 方法默认public abstract - 实现类必须实现所有抽象方法

方法的参数传递

Java 中方法参数传递只有值传递。

基本数据类型作为参数：传递值的拷贝，方法内修改不影响实参。

    static void change(int num) {


        num = 100;


    }




    public static void main(String[] args) {


        int a = 10;


        change(a);

    }


}

引用数据类型作为参数：传递引用（地址）的拷贝。 - 通过引用修改对象属性会影响原对象 - 修改引用本身（指向新对象）不影响原引用

class Person {


    String name;


    Person(String name) {


        this.name = name;


    }


}




public class Test {


    static void changeName(Person p) {


        p.name = "李四";


    }




    static void changeNew(Person p) {


        p = new Person("王五");


    }




    public static void main(String[] args) {


        Person p = new Person("张三");


        changeName(p);


        System.out.println(p.name); // 李四




        changeNew(p);


        System.out.println(p.name); // 李四


    }


}

异常处理

  • 异常定义：程序运行过程中发生的意外事件，中断正常执行流程。

异常分类结构

Throwable
├── Error：严重错误，程序无法处理（如 OutOfMemoryError）
└── Exception：程序可以处理的异常
    ├── RuntimeException（运行时异常/非检查型异常）
    └── 非RuntimeException（检查型异常/编译时异常）

常见异常类及含义

异常类名 含义说明
NullPointerException 访问 null 对象的成员
ArrayIndexOutOfBoundsException 数组索引越界
ArithmeticException 算术错误（如除以 0）
ClassCastException 类型转换错误
NumberFormatException 数字格式转换错误
IOException I/O 操作失败
FileNotFoundException 文件未找到
SQLException 数据库相关异常
  • 运行时异常：不需要强制捕获，程序运行时才可能出现
  • 编译时异常：必须显式处理（try-catch 或 throws），否则编译不通过

基本结构

try {


    // 可能发生异常的代码


} catch (异常类型1 变量名) {


    // 处理异常类型1


} catch (异常类型2 变量名) {


    // 处理异常类型2（先子类后父类）


} finally {


    // 无论是否发生异常都会执行（释放资源）


}
关键字 作用说明
try 包裹可能发生异常的代码块
catch 捕获并处理指定类型异常，多个 catch 需先子类后父类
finally 始终执行，常用于关闭流、释放连接，即使有 return 也会执行
throw 在方法内部手动抛出一个异常对象，执行后当前代码块终止
throws 在方法声明处标识可能抛出的异常，交给上层调用者处理

throw 与 throws 区别 - throw：在方法体内使用，抛出一个异常对象 - throws：在方法签名后使用，声明可能抛出的异常类型 - throw 立即产生异常；throws 只是声明，不立即抛出

代码示例：

public class ExceptionDemo {


    public static void main(String[] args) {


        try {


            int a = 10;


            int b = 0;


            int c = a / b;


            System.out.println(c);


        } catch (ArithmeticException e) {


            System.out.println("算术异常：不能除以0");


        } catch (Exception e) {


            System.out.println("其他异常");


        } finally {


            System.out.println("finally 始终执行");


        }


    }


}

文件与数据流

基本概念： - 输入流：从数据源（文件、网络等）读取数据到程序（读操作） - 输出流：从程序写出数据到目标（文件、网络等）（写操作）

字节流 vs 字符流： - 字节流：以字节(byte)为单位，用于处理二进制数据（图片、视频等） - 字符流：以字符(char)为单位，用于处理文本数据

核心流类

基础流类 - InputStream：所有字节输入流的父类（抽象类） - OutputStream：所有字节输出流的父类（抽象类）

文件流类 - FileInputStream：从文件读取字节的输入流 - FileOutputStream：向文件写入字节的输出流

数据流类 - DataInputStream：从底层输入流读取基本数据类型 - DataOutputStream：向底层输出流写入基本数据类型

缓冲流类 - BufferedInputStream：带缓冲区的字节输入流，提高读效率 - BufferedOutputStream：带缓冲区的字节输出流，提高写效率

常用方法：

方法 功能说明
read() 读取一个字节，返回 int，-1 表示结束
write(int b) 写入一个字节
readInt() 读取一个 int（4 字节）
writeInt(int v) 写入一个 int（4 字节）
seek(long pos) 将文件指针定位到指定位置
skipBytes(int n) 跳过 n 个字节

字节流文件复制（缓冲流）：

import java.io.BufferedInputStream;


import java.io.BufferedOutputStream;


import java.io.FileInputStream;


import java.io.FileOutputStream;


import java.io.IOException;




public class FileCopyDemo {


    public static void main(String[] args) {


        try {


            FileInputStream fis = new FileInputStream("source.dat");


            BufferedInputStream bis = new BufferedInputStream(fis);




            FileOutputStream fos = new FileOutputStream("target.dat");


            BufferedOutputStream bos = new BufferedOutputStream(fos);




            int b;


            while ((b = bis.read()) != -1) {


                bos.write(b);


            }




            bis.close();


            bos.close();


        } catch (IOException e) {


            System.out.println("文件操作异常");


        }


    }


}
